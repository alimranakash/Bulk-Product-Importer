Why Bulk Product Importer?

Managing a WooCommerce store with hundreds or thousands of products is challenging. Manual entry takes forever, and the built-in CSV importer often fails with large files. Bulk Product Importer solves these problems with:

  • Simple Excel Format: Use familiar .xlsx files instead of complex CSV formatting
  • Image Bundling: Include all product images in the same ZIP file
  • Background Processing: Imports run in the background using WooCommerce's Action Scheduler
  • No Timeouts: Products are processed in small batches, eliminating timeout errors
  • Variable Products: Full support for variable products with unlimited variations
  • Smart Updates: Existing products are updated based on SKU matching

üîß Troubleshooting

Common Issues & Solutions

1. "ZIP file could not be extracted"

üî¥
Problem

The ZIP file is corrupted or the PHP zip extension is not installed.

Solution:

  • Verify ZIP file opens correctly on your computer
  • Check PHP zip extension: php -m | grep zip
  • Contact hosting to enable zip extension if missing

2. "No Excel files found in ZIP"

Causes:

  • Excel files are in a subfolder instead of root
  • File extension is not .xlsx or .xls
  • Files are hidden/system files

Solution: Place Excel files directly in ZIP root, not in subfolders.

3. "Images not appearing on products"

Causes:

  • Images folder not named exactly images (case-sensitive)
  • Filenames in Excel don't match actual files
  • Image files are in subfolders within images folder

Solution:

# Correct structure:
product-import.zip
‚îú‚îÄ‚îÄ products.xlsx
‚îî‚îÄ‚îÄ images/          # Must be lowercase "images"
    ‚îú‚îÄ‚îÄ photo1.jpg   # Files directly in images folder
    ‚îî‚îÄ‚îÄ photo2.jpg

4. "Import stuck at 0%"

Causes:

  • WP-Cron is disabled or not working
  • Action Scheduler is broken
  • Server resource limits hit

Solution:

// Check if DISABLE_WP_CRON is set in wp-config.php
define('DISABLE_WP_CRON', false);  // Should be false or not set

// Alternatively, trigger cron manually:
curl https://your-site.com/wp-cron.php

5. "Memory limit exhausted"

Solution: Add to wp-config.php:

define('WP_MEMORY_LIMIT', '256M');
define('WP_MAX_MEMORY_LIMIT', '512M');

Also reduce batch size in plugin settings to 10 or 25.

6. "Variations not linking to parent"

Causes:

  • Parent SKU in variation doesn't match
  • Parent product processed after variations
  • Missing attribute values

Solution: Ensure parent products appear BEFORE variations in Excel, and Parent SKU matches exactly.

‚ùì Frequently Asked Questions

Q: Can I import products while the store is live?

A: Yes! Background processing ensures your store remains responsive. However, for very large imports (10,000+), we recommend doing it during low-traffic hours to minimize any potential impact.

Q: Will importing overwrite my existing products?

A: Products are matched by SKU. If a product with the same SKU exists, it will be updated with the new data. If the SKU doesn't exist, a new product is created.

Q: Can I import products with existing images URLs instead of local files?

A: The current version focuses on local image bundling for reliability. URL-based imports often fail due to hotlinking restrictions. We recommend downloading images and including them in the ZIP.

Q: How do I create a variable product with color and size options?

A: Use multiple attribute columns:

Type: variable
SKU: SHIRT-001
Attribute 1 Name: Color
Attribute 1 Values: Red|Blue|Green
Attribute 2 Name: Size
Attribute 2 Values: S|M|L|XL

Then create variations for each combination.

Q: What happens if the import fails midway?

A: Products already imported remain in WooCommerce. You can re-run the import ‚Äì existing products (matching SKU) will be updated, not duplicated.

Q: Can I schedule imports to run automatically?

A: The current version requires manual upload. Automatic scheduled imports from external sources (FTP, URLs) is planned for a future release.

Q: Is there a limit on ZIP file size?

A: The limit is determined by your PHP/server settings (upload_max_filesize, post_max_size). We've tested with ZIP files up to 500MB containing 20,000 products.

Q: Do I need WooCommerce installed?

A: Yes, WooCommerce must be installed and activated. The plugin creates WooCommerce products using the official API.

Q: Can I import product reviews and ratings?

A: Not in the current version. Product reviews import is on our roadmap.

Q: Does it work with WPML for multilingual stores?

A: Basic compatibility is supported. Import products in your default language first, then use WPML's translation tools. Native multi-language import is planned.

‚úÖ Best Practices

Preparing Your Data

  • Use consistent SKU format: CATEGORY-PRODUCT-VARIANT (e.g., SHIRT-POLO-BLUE-L)
  • Clean your data: Remove extra spaces, check for special characters
  • Validate before import: Open Excel and check all rows have required fields
  • Use UTF-8: For international characters (√º, √±, Êó•Êú¨Ë™û), ensure Excel saves as UTF-8

Image Optimization

  • Resize images: 1000x1000px is optimal for most themes
  • Compress before ZIP: Use TinyPNG or similar to reduce file sizes
  • Use descriptive names: blue-cotton-shirt-front.jpg not IMG_3847.jpg
  • Avoid duplicates: If multiple products share an image, it's OK to reference the same filename

Large Imports (5000+ products)

  • Split into batches: Create multiple ZIPs of 1000-2000 products each
  • Import overnight: Start large imports before leaving for the day
  • Monitor resources: Keep an eye on server memory and disk space
  • Disable emails: Consider temporarily disabling order notification emails during import

After Import

  • Verify products: Spot-check 10-20 products for accuracy
  • Check images: Ensure featured images and gallery images are correct
  • Test variations: Add variable products to cart and test variation selection
  • Clear caches: Flush any caching plugins and CDN caches
