Excel File Overview

The heart of Bulk Product Importer is your Excel spreadsheet. This file contains all the information about your products ‚Äì names, prices, descriptions, images, categories, and more. Getting the format right is crucial for a successful import.

Supported File Formats

  • .xlsx ‚Äì Modern Excel format (recommended)
  • .xls ‚Äì Legacy Excel format

Basic Structure

Your Excel file should follow this basic structure:

  • Row 1: Column headers (must match exactly)
  • Row 2+: Product data (one product per row)

üí° Pro Tip

The easiest way to get the correct format is to export a few products from WooCommerce first. Go to Products ‚Üí All Products ‚Üí Export and use that file as a template.

Attribute Columns

The plugin supports up to 3 attributes per product. Each attribute requires 4 columns:

Column Header Required Description & Example
Attribute 1 name Optional Attribute name. Example: Size or Color
Attribute 1 value(s) Optional Values separated by comma. Example: S, M, L, XL
Attribute 1 visible Optional Show on product page: 1 or 0. Default: 1
Attribute 1 global Optional Use global attribute: 1 or 0. Default: 1

Replace 1 with 2 or 3 for additional attributes.

Simple Products

Simple products are the most straightforward to import. Each row represents one complete product.

Example: Simple Products

Type SKU Name Regular price Sale price Categories Images
simple BOOK-001 WordPress Development Guide 49.99 39.99 Books > Programming book-wp-guide.jpg
simple MUG-001 Developer Coffee Mug 14.99 Accessories > Mugs mug-dev.jpg, mug-dev-2.jpg
simple STICKER-001 Laptop Sticker Pack 9.99 7.99 Accessories > Stickers stickers-pack.jpg

‚úÖ Key Points for Simple Products

  • Type must be simple
  • SKU must be unique across all products
  • Leave Sale price empty if there's no discount
  • Multiple images are separated by commas

Variable Products & Variations

Variable products are more complex but incredibly powerful. They allow you to sell products with different options like sizes, colors, or materials.

Understanding the Structure

A variable product consists of:

  1. Parent Product (Type = variable) ‚Äì Defines the product and its attributes
  2. Variations (Type = variation) ‚Äì Individual purchasable items with specific attribute values

Example: T-Shirt with Size and Color

Let's create a T-Shirt available in 3 sizes (S, M, L) and 2 colors (Blue, Red):

Step 1: Parent Product Row

Type SKU Name Attr 1 name Attr 1 value(s) Attr 2 name Attr 2 value(s)
variable TSHIRT-001 Premium Cotton T-Shirt Size S, M, L Color Blue, Red

Step 2: Variation Rows

Type SKU Parent Regular price Attr 1 name Attr 1 value(s) Attr 2 name Attr 2 value(s)
variation TSHIRT-001-S-BLU TSHIRT-001 24.99 Size S Color Blue
variation TSHIRT-001-M-BLU TSHIRT-001 24.99 Size M Color Blue
variation TSHIRT-001-L-BLU TSHIRT-001 26.99 Size L Color Blue
variation TSHIRT-001-S-RED TSHIRT-001 24.99 Size S Color Red
variation TSHIRT-001-M-RED TSHIRT-001 24.99 Size M Color Red
variation TSHIRT-001-L-RED TSHIRT-001 26.99 Size L Color Red

‚ö†Ô∏è Important Notes

  • The Parent column must contain the exact SKU of the variable product
  • Each variation should have only ONE value per attribute (not comma-separated)
  • Variable products are processed before variations (the plugin sorts automatically)
  • Variation prices can differ from each other

Image Configuration

There are two ways to add images to your products:

Option 1: Images in ZIP File (Recommended)

Place your images in an images folder inside your ZIP. Then reference them by filename in Excel:

Images column: product-001.jpg, product-001-side.jpg, product-001-back.jpg

The first image becomes the featured image; additional images go to the gallery.

Option 2: External URLs

You can also use full URLs to images hosted elsewhere:

Images column: https://example.com/images/product-001.jpg

üìÅ Supported Image Formats

  • JPG/JPEG ‚Äì Best for photos
  • PNG ‚Äì Best for graphics with transparency
  • WebP ‚Äì Modern format with excellent compression
  • GIF ‚Äì For simple animations

Categories & Attributes

Category Format

Use > to create hierarchical categories:

Clothing > Men > T-Shirts

This creates a nested structure: Clothing ‚Üí Men ‚Üí T-Shirts. Categories are auto-created if they don't exist.

Multiple Categories

Assign a product to multiple categories using a pipe |:

Clothing > Men | Featured | Sale Items

Common Mistakes to Avoid

‚ùå Common Errors

  • Empty SKU: All products (except variations) must have a unique SKU
  • Wrong Parent SKU: Variations must reference the exact parent SKU
  • Spaces in Image Names: Use hyphens or underscores instead of spaces
  • Wrong Column Headers: Headers must match exactly (case-sensitive)
  • Missing Attributes: Variable products must define all attributes used by variations
  • Variation Before Parent: Variable products must be listed before their variations

Pro Tips & Best Practices

  • Export First: Export existing products from WooCommerce to see the correct format
  • Test Small: Import 10-20 products first to verify format
  • Use Consistent SKUs: Create a SKU naming convention (e.g., CATEGORY-ITEM-VARIANT)
  • Optimize Images: Resize images to 1000x1000px max before including in ZIP
  • Split Large Files: For 10,000+ products, split into multiple Excel files
  • Check Encoding: Save Excel files with UTF-8 encoding for special characters
